Core requirements (required)
Responsive card grid — each card shows cover image, title, category, location, price, and rating. Layout: 3 columns on desktop, 2 on tablet, 1 on mobile. Mobile-first.
Category filter — filter cards by category (Tours, Outdoor, Food & Drink, Family), plus an "All" reset. Active filter clearly highlighted.
Search — text input filters cards live by title as the user types; works together with the category filter.
Details modal — clicking a card opens a modal with all images, full description, price, rating, and location. Must include a working image carousel/slider (Swiper or Slick). Closes via X button, backdrop click, and ESC.
Empty + loading states — a friendly "No results found" message when nothing matches, and a brief loading state before cards render.
Bonus (optional)
A "Request Info" lead form in the modal (name, email, phone) with client-side validation and a success message (simulate submit, no backend).
Lazy-load card images.
A small Google Map (or static map placeholder) in the modal showing the location.
Sort by price or rating.
